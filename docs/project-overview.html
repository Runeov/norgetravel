1. What is This Project?

Averdi is a professional website for a state-authorized accounting firm based in Karasjok, Northern Norway. The company positions itself as "The Interpreter of Northern Norway" (Tolken av Nord-Norge).

Specializations

  • Accounting services for Northern Norwegian businesses
  • Sami Parliament (Sametinget) grant navigation
  • Tax advantages in the "Tiltakssonen" (Action Zone)
  • Advisory services for regional incentives

5. Admin Panel Features

👥 Employee Management

Add, edit, and remove team members. Control display order and active status.

📝 Article Editor

Rich text editor (TipTap) with formatting, images, and links.

📊 Dashboard

Overview of employees, published articles, and drafts.

🔐 Authentication

Password-protected with JWT session cookies.

Employee Management Capabilities

  • View all employees — List with active/inactive status
  • Create new employees — Add name, role, bio, specialties, image
  • Edit existing employees — Update any field
  • Delete employees — Remove from the system
  • Reorder employees — Control display order on public site

Article Management Capabilities

  • Create articles — Rich text editor with full formatting
  • Draft/Publish workflow — Save drafts before publishing
  • Categories — Organize by topic (bedrift, sametinget, etc.)
  • Featured articles — Highlight important content

6. Authentication Flow

Login Process
1. User visits /admin
↓
2. Redirected to /admin/login
↓
3. Enter password
↓
4. API verifies credentials against users.json + bcrypt hash
↓
5. JWT cookie set, redirect to dashboard

Security Note: The admin panel uses HTTP-only cookies with JWT tokens. Sessions expire after 24 hours. Passwords are stored as bcrypt hashes (users.json), with optional hash override via environment variables.

9. Running the Project

Development

# Install dependencies
npm install

# Start development server
npm run dev

# The site will be available at http://localhost:3000

Production

# Build for production
npm run build

# Start production server
npm start

Environment Variables

Create a .env.local file in the project root:

# Required for admin panel
JWT_SECRET=your-secure-jwt-secret

# Optional initial-admin password hash override
ADMIN_PASSWORD_HASH=your-bcrypt-hash-here

# Optional analytics
NEXT_PUBLIC_HOTJAR_ID=your-hotjar-site-id
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
